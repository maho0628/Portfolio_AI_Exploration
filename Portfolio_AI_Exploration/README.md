# Manual Override

プレイヤー介入型AIを用いた探索アドベンチャー（Planning / Prototype）

---

## 概要

**Manual Override** は、  
「AIに任せる行動」と「プレイヤーが操作で割り込む瞬間」の境界をテーマにした、  
探索×操作介入型アドベンチャーゲームのプロトタイプです。

キャラクターは基本的に自動で行動しますが、  
特定のタイミングでプレイヤー入力による介入が可能となる設計を採用しています。

本プロジェクトでは、
最終的にプレイ可能な作品として完成させることを前提としつつ、
AI設計および挙動設計の検証・言語化を重視して制作しています。

---

## 制作目的

- AI制御とプレイヤー操作が共存する構造の検証  
- 行動処理と状態遷移を分離したAI設計の実装  
- 後から調整・拡張しやすい、データ駆動型構成の検討  

「賢いAI」を作ることではなく、  
**“扱いやすく、説明できるAI構造”** を目指しています。

---

## ゲーム構成（予定）

### 探索パート

- 3Dフィールドを自由に移動
- NavMesh を用いた簡易的な移動・巡回AI
- 敵や探索対象の発見によって戦闘パートへ遷移

### 戦闘パート

- キャラクターは基本的に自動で行動
- 行動ゲージが溜まるとスキル準備状態へ遷移
- プレイヤー入力によりスキルを発動、または待機を選択

---

## AI設計方針

- 行動ルールは全キャラクター共通
- キャラクターごとの差異はデータで制御
- 行動処理と状態遷移を明確に分離
- プレイヤー入力は「割り込みイベント」として扱う

AIは最適解を導く存在ではなく、  
**状態管理と操作受付を担う存在** として設計しています。

---

## 開発フローについて

本プロジェクトでは、  
挙動検証や実験は `_Dev/TestScenes` 以下で行います。

仕様や設定値が確定した要素のみを、  
本番用シーン（例：`Exploration_Main`）へ反映する運用としています。

Prototype シーンは破棄前提、  
本番シーンは将来差し替えない前提で扱います。

---

## 探索パート設計（NavMesh）

探索パートでは Unity の NavMesh を用いてキャラクター移動を制御します。

Agent の半径や障害物との距離感は、  
Prototype シーン上で検証を行い、  
到達不能時の挙動も含めて設計方針を決定しています。

本段階では仮シーン上で挙動確認を行い、  
確定した設定のみを本番シーンへ転記します。

---

## Git / ブランチ運用

機能追加や検証はブランチを分けて行い、  
Prototype や素材検証段階の変更は即時 main へは統合しません。

設計や方向性が固まった段階で  
develop → main へ反映する運用としています。
